# Security Policy

## Supported Versions

Security fixes are applied to the latest code on `master`.

## Reporting a Vulnerability

If you discover a security issue, do not open a public issue with exploit details.

Please report privately with:

- A clear description of the issue
- Impact and affected components
- Reproduction steps or proof of concept
- Suggested mitigation if available

Use repository private contact channels or maintainers' direct contact.

## Scope

High-priority issues include:

- API key/token exposure
- Unauthorized trade execution paths
- Authentication/authorization bypasses
- Command injection or unsafe external input handling
- Sensitive data leaks in logs or reports

## Response Targets

- Initial acknowledgement: within 72 hours
- Triage decision: within 7 days
- Patch timeline: based on severity and exploitability
